What is AOS?

An operating system layer for macOS that turns a Mac Mini into an autonomous workstation. AI agents manage your work, run tasks, compound knowledge, and improve over time.

AOS doesn't build an agent framework. It configures Claude Code as its runtime with CLAUDE.md files, agent definitions, skills, hooks, and persistent state. Claude Code is the engine. The filesystem is memory.

The system works at three scales. You start where you are and grow into the rest.

UserWhat they get
Solo personA teacher, chef, or freelancer. One machine, one place for everything. Add a task, see your tasks. That's it.
Multi-project3 businesses, 7 projects. Visibility across all of them. Agents handling the routine work.
Full agenticAutonomous agents running parts of businesses. Trust ramp. Drift detection. Self-improving system.

What you get

Chief is your AI orchestrator. Every request enters through Chief. It decides whether to handle things directly or dispatch to a specialist agent. Think of it as the init process of your OS.

Work System tracks tasks, projects, and goals automatically. Sessions link to tasks. Repeated patterns compile into scripts. Reviews generate themselves. Like Git is infrastructure for code, this is infrastructure for work.

Knowledge Vault captures everything you learn. Hybrid search (BM25 + semantic + reranking) makes it all retrievable. Obsidian provides the GUI. The system gets smarter every day.

Dashboard is the web UI you're looking at right now. Activity feed, work tracking, agent registry, session history, and logs. All on localhost:4096.

Integrations connect Telegram, WhatsApp, email, calendar, and more. Each has a manifest, setup script, and health check. Configured during onboarding or added later.

Quick start

curl -fsSL https://raw.githubusercontent.com/agentalhadi/aos/main/install.sh | bash

Idempotent. Safe to re-run. Takes about 5 minutes. When it finishes, type claude and Chief will guide you through onboarding.

Core concepts

Claude Code as runtime

Claude Code is already an agent framework. AOS doesn't build a new one. It configures the existing one with CLAUDE.md, agents, skills, hooks, and state. Like configuring Linux with dotfiles — the tool is the same, the configuration makes it yours.

Chief as init process

Every request enters through Chief. Chief decides: handle directly or dispatch to a specialist. Subagents run in isolated context via the Agent tool. State is shared through the filesystem, not message passing.

Filesystem as persistent memory

Context window is RAM — volatile, limited. The filesystem is disk — persistent, unlimited. Anything important gets written to files. Config YAML, vault notes, execution logs, work state. Git gives you history for free.

Five loops drive the system:

Chief: Request → Decision → Dispatch → Response
Steward: Monitor → Detect → Correct → Report
Advisor: Observe → Pattern → Compile → Recommend

Design decisions

DecisionRationale
Claude CodeRuns on your subscription. No API keys. Built-in subagents, file/bash/web tools, headless mode for daemons.
macOS KeychainNative, hardware-backed encryption. No external dependencies.
Localhost onlyZero attack surface. Tailscale for authenticated remote access.
File-based stateNo database to manage. The filesystem is the database. Git gives history. Agents and humans read the same files.
No orchestration libClaude Code's subagent system handles dispatch. Adding LangGraph etc. would add complexity without proportional benefit.
Symlink system, copy catalogSystem agents auto-update with the OS. Catalog agents are copied so user customizations survive updates.

System agents

Chief
core/agents/chief.md · opus · required
The brain. Orchestrator. Receives all requests, decides to handle directly or dispatch to a specialist. Always active — this is the agent you talk to when you type claude.
Steward
core/agents/steward.md · haiku
The immune system. Health monitoring, self-correction, drift detection, system repairs. Runs on haiku for speed — you want health checks to be fast, not philosophical.
Advisor
core/agents/advisor.md · sonnet
The nervous system. Analysis, knowledge curation, work planning, reviews. Observes patterns, compiles insights, surfaces what matters. Runs on sonnet for nuanced analysis.

Why 3 system agents, not 5? Analyst, Curator, and Planner were considered as separate agents but merged into Advisor. On day one, there isn't enough knowledge to curate or work to plan. One agent with multiple skills starts lean. Split when evidence shows a function needs its own trust level.

What's not an agent

Deterministic work runs as cron jobs: session-export, pattern-compile, log-rotate, health-check. The test: "Does it take autonomous actions that need a graduated trust level?" If no, it's a script.

Trust ramp

Trust is per-capability, not per-agent. An agent might be fully trusted for file operations but require approval for sending messages.

LevelNameBehavior
0ShadowObserve only — log what it would do, don't execute
1ApprovalAgent proposes, operator approves each action
2Semi-autoAct on high confidence (>0.85), ask on uncertain
3Full-autoHandle everything, escalate exceptions only

Graduation is weighted: routine actions count 0.5, novel actions 2.0, complex actions 3.0. A revert costs -5.0. Trust is earned, not given.

Where agents live on disk

~/aos/core/agents/           Source for system agents
~/aos/templates/agents/      Catalog templates
~/.claude/agents/            Active (installed, available everywhere)
~/project/.claude/agents/    Project-scoped agents

System agents are symlinked (auto-update with the OS). Catalog agents are copied so user customizations survive updates. Catalog agents track their source via frontmatter: _source: catalog/engineer@1.0.

Work System

Infrastructure for managing work. Like Git is infrastructure for code, this is infrastructure for work.

Goals → Projects → Tasks → Sessions → Knowledge → Reviews → Goals

Design principles

  • Useful on day 1. Add a task, see your tasks. That's it. Complexity is opt-in.
  • File-first. Structured files in a directory. No database. Git gives you history for free.
  • Agent-native, human-friendly. Agents are first-class workers. But the system works perfectly with zero agents.
  • Progressive complexity. Start with a flat task list. Add projects when you need them. Goals when you're ready. Agents when you trust them.
  • Minimum viable overhead. If maintaining the system takes more than 5 minutes a day, it's too complex.

Task IDs

Tasks use project-scoped IDs: aos#3, chief#1, t#1 (unassigned). Subtasks use dot notation: aos#3.1, aos#3.2.

Fuzzy matching resolves titles, so work done "landing page" finds the right task without needing the exact ID.

Services

Always-on processes managed by macOS LaunchAgents. They survive reboots and run independently of Claude sessions.

Bridge
daemon · core/services/bridge/
Telegram messaging, forum topic routing, heartbeat every 30 minutes, daily briefing at 8am, voice transcription, and Claude CLI dispatch. The primary way to talk to the system remotely.
Dashboard
:4096 · core/services/dashboard/
This web UI. Agent registry, real-time activity feed via SSE, work tracking, session history, log viewer, and system health monitoring.
Listen
:7600 · core/services/listen/
HTTP job server. Receives tasks via API, spawns Claude Code worker processes, and tracks job state via YAML. The background execution engine.
Memory
stdio · core/services/memory/
Semantic search via ChromaDB. An MCP server that any Claude session can query for file recall and context retrieval.

Execution model

Claude Code is the execution engine. AOS does not use LangGraph, Paperclip, or any external orchestration framework.

TriggerWhat happens
Telegram messageBridge dispatches to claude -p with project-specific --cwd and --agent
HTTP jobListen spawns a Claude Code worker, tracks via YAML
Agent invocationClaude Code's built-in subagent system (.claude/agents/*.md)
Memory queryMCP server exposes semantic search to any active session

Networking

All services bind to 127.0.0.1. No ports are exposed to the network. Remote access is exclusively through Tailscale — an authenticated, encrypted mesh VPN.

  • Zero attack surface from the internet
  • Access from your phone or laptop via Tailscale IP
  • Each device is authenticated — no passwords to manage

Rules

  • No user data in the system repo. A git clean -fd on ~/aos/ must never destroy user data.
  • Secrets live in Keychain only. Never in files. agent-secret get/set wraps macOS Keychain.
  • ~/aos/ is read-only at runtime. It's the framework. Updates ship as versioned releases.
  • ~/.aos/ is machine-specific. Never committed. Contains all the state that makes this installation unique.
  • Vault is independent. It existed before AOS. Obsidian opens it by path. QMD indexes it independently. The path is configurable.
